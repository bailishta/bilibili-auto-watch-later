# 稍后观看助手

自动将关注的 B站 UP主发布的新视频添加到「稍后观看」列表的 Chrome 扩展。

## 功能

- **自动检查新视频** — 定时扫描追踪名单中 UP主的最新视频，自动添加到稍后观看
- **一键导入关注** — 首次使用时一键导入 B站关注列表中的所有 UP主
- **手动管理名单** — 支持手动添加/移除 UP主，灵活控制追踪范围
- **观看历史过滤** — 自动跳过已经观看过的视频，避免重复添加
- **可配置时间窗口** — 支持设定「多久以内的视频算新视频」（1天~7天）

## 安装

1. 下载本仓库的 [最新 Release](https://github.com/your/repo/releases) 中的 `.zip` 文件并解压
2. 打开 Chrome，进入 `chrome://extensions/`
3. 开启右上角**「开发者模式」**
4. 点击**「加载已解压的扩展程序」**，选择解压后的文件夹
5. 安装完成后会自动打开引导页，可一键导入 B站关注列表

## 使用说明

- 点击扩展图标打开弹窗，可查看状态、手动触发检查
- 弹窗中可添加/移除追踪的 UP主（支持 UID 或主页链接）
- 设置「新视频判定」时间窗口和「自动检查间隔」
- 点击「打开稍后观看」可跳转到 B站稍后观看列表

## 注意事项

- 需要登录 B站（确保浏览器中已登录 bilibili.com）
- 自动检查依赖 Chrome 后台定时器，如果长时间未打开 Chrome 可能会延迟
- 已观看的视频会被自动跳过，不会重复添加到稍后观看
- 仅支持 Chrome 浏览器（基于 Manifest V3）

## 项目结构

```
├── manifest.json          # Chrome 扩展配置
├── service-worker.js      # 后台服务（定时触发、消息路由）
├── content.js             # 内容脚本（B站页面代理请求）
├── core/
│   ├── api.js             # B站 API 封装（WBI签名、Cookie、请求）
│   ├── checker.js         # 核心业务（扫描新视频、导入关注）
│   └── storage.js         # chrome.storage 封装
├── popup/                 # 扩展弹窗 UI
├── onboarding/            # 首次安装引导页
└── icons/                 # 扩展图标
```

纯 JavaScript + Manifest V3，无构建步骤，无外部依赖。
